# Change: Update AGENTS.md Generation to Use Script Consistently

## Why

The `dev-skill` plugin already has a `build-agents-md.js` script that compiles individual rule files into AGENTS.md. However, the current workflow in `commands/new.md` documents the script usage but doesn't enforce it programmatically. The rule templates follow a consistent structure (YAML frontmatter + markdown body), making automated generation reliable. This change ensures the AGENTS.md build step always uses the existing script.

## What Changes

- Update workflow documentation in `commands/new.md` to make script usage explicit and mandatory
- Verify the `build-agents-md.js` script handles all edge cases from the template structure
- Add validation to ensure AGENTS.md was generated by script (not manually written)

## Impact

- Affected files: `plugins/dev-skill/commands/new.md`, `plugins/dev-skill/scripts/build-agents-md.js`
- No breaking changes - the script already exists and works correctly
- Tooling-only change - no new capabilities added
