# Skill Structure Migration Command

Migrate skills generated by dev-skill from the old `rules/` directory structure to the new `references/` + `assets/templates/` structure with 100% validation to eliminate breaking changes.

## Input

A single skill directory path, e.g.:
- `~/.claude/skills/vercel-react-best-practices`
- `./skills/python-best-practices`

## Structure Change Summary

### Old Structure
```
skill/
├── SKILL.md
├── AGENTS.md
├── metadata.json
├── README.md
└── rules/
    ├── _sections.md
    ├── _template.md
    └── {prefix}-{slug}.md
```

### New Structure
```
skill/
├── SKILL.md              # Updated with markdown links
├── AGENTS.md             # Regenerated with Source Files footer
├── metadata.json
├── README.md
├── references/
│   ├── _sections.md
│   └── {prefix}-{slug}.md
└── assets/
    └── templates/
        └── _template.md
```

---

## Migration Workflow

Execute these steps in order. Content is MOVED, not rewritten, except for necessary link updates.

### Step 1: Pre-flight Checks

Before any modifications:

```bash
# 1. Verify skill path exists
test -d "<skill-path>" || { echo "Error: Skill directory not found"; exit 1; }

# 2. Verify old structure exists (needs migration)
test -d "<skill-path>/rules" || { echo "Error: No rules/ directory - skill may already be migrated"; exit 1; }

# 3. Verify NOT already migrated
test ! -d "<skill-path>/references" || { echo "Error: references/ exists - already migrated"; exit 1; }

# 4. Check if git repo and content is committed
cd "<skill-path>"
if git rev-parse --git-dir > /dev/null 2>&1; then
  # Check if skill files are committed (no uncommitted changes)
  if ! git diff --quiet . && ! git diff --cached --quiet .; then
    echo "Warning: Uncommitted changes detected in skill directory"
    # Ask user to commit first or proceed with caution
  fi
fi
```

**If pre-flight fails**: Report the specific error and stop.

### Step 2: Capture Before State

Before making changes, capture state for validation:

```bash
# List all rule files (excluding _ prefixed)
ls -1 "<skill-path>/rules/"*.md | grep -v '/_' > /tmp/rules_before.txt

# Count rules
RULE_COUNT_BEFORE=$(ls -1 "<skill-path>/rules/"*.md 2>/dev/null | grep -v '/_' | wc -l | tr -d ' ')

# Capture AGENTS.md line count
AGENTS_LINES_BEFORE=$(wc -l < "<skill-path>/AGENTS.md" | tr -d ' ')

# Capture checksums for content verification
find "<skill-path>/rules" -name "*.md" -type f -exec md5 {} \; > /tmp/checksums_before.txt
```

### Step 3: File Migration (Move, Don't Rewrite)

Execute moves in this order:

```bash
cd "<skill-path>"

# 1. Create new directories
mkdir -p references
mkdir -p assets/templates

# 2. Move _sections.md to references/
mv rules/_sections.md references/_sections.md

# 3. Move all rule files to references/
# (all .md files except _template.md)
for f in rules/*.md; do
  [ "$(basename "$f")" = "_template.md" ] && continue
  mv "$f" references/
done

# 4. Move _template.md to assets/templates/
mv rules/_template.md assets/templates/_template.md

# 5. Remove empty rules/ directory
rmdir rules
```

**Content preservation**: Files are moved (mv), not copied or rewritten. Only metadata (path) changes.

### Step 4: Update SKILL.md Links

Read SKILL.md and update references. This is the only content rewriting needed:

**Path replacements**:
- `rules/` -> `references/`
- `rules/_sections.md` -> `references/_sections.md`
- `rules/_template.md` -> `assets/templates/_template.md`

**Link format conversion** (where applicable):
- Plain text `rules/example.md` -> `[example](references/example.md)`
- Ensure "How to Use" section uses markdown link format

Example transformation:
```markdown
# Before
Read individual rule files:
rules/{prefix}-{slug}.md
rules/_sections.md

# After
Read individual reference files:
- [Section definitions](references/_sections.md) - Category structure and impact levels
- [Rule template](assets/templates/_template.md) - Template for adding new rules
```

Use Read tool to get SKILL.md content, apply transformations, then Write tool to save.

### Step 5: Regenerate AGENTS.md

Use the build script which already supports the new structure:

```bash
node "${CLAUDE_PLUGIN_ROOT}/scripts/build-agents-md.js" "<skill-path>"
```

The script will:
- Read from `references/` directory
- Generate "Source Files" footer with navigation links
- Maintain all rule content

### Step 6: Validation Pipeline

Run all validation checks. ALL must pass.

#### 6.1 Structural Validation

```bash
cd "<skill-path>"

# All rule files exist in references/
RULE_COUNT_AFTER=$(ls -1 references/*.md 2>/dev/null | grep -v '/_' | wc -l | tr -d ' ')
[ "$RULE_COUNT_BEFORE" = "$RULE_COUNT_AFTER" ] || echo "FAIL: Rule count mismatch"

# _sections.md exists in references/
test -f references/_sections.md || echo "FAIL: _sections.md not found"

# _template.md exists in assets/templates/
test -f assets/templates/_template.md || echo "FAIL: _template.md not found"

# rules/ directory removed
test ! -d rules || echo "FAIL: rules/ directory still exists"
```

#### 6.2 Content Validation

```bash
# No rules/ references remain in SKILL.md
grep -q "rules/" SKILL.md && echo "FAIL: SKILL.md still contains rules/ references"

# SKILL.md contains references/ links
grep -q "references/" SKILL.md || echo "FAIL: SKILL.md missing references/ links"
```

#### 6.3 Build Validation

```bash
# validate-skill.js passes
node "${CLAUDE_PLUGIN_ROOT}/scripts/validate-skill.js" "<skill-path>"
# Must exit with 0 errors

# AGENTS.md was regenerated successfully (exists and has content)
test -s AGENTS.md || echo "FAIL: AGENTS.md empty or missing"
```

#### 6.4 Content Preservation Check

```bash
# Compare file checksums (content should be identical for moved files)
# Note: AGENTS.md will differ (regenerated), SKILL.md will differ (links updated)
# Rule files in references/ should have same content as rules/ did
```

### Step 7: Migration Judge Review

Invoke the `migration-judge` agent to review the migration:

```
Use the Task tool with subagent_type: "dev-skill:migration-judge"

Prompt: Review the migration of skill at "<skill-path>".
Verify:
1. All rule files were moved correctly (content preserved)
2. SKILL.md links updated appropriately
3. AGENTS.md regenerated with all rules
4. No breaking changes introduced

Provide PASS/FAIL verdict with reasoning.
```

**If migration-judge returns FAIL**: Report issues to user.

---

## Error Handling

If migration fails at any step:

1. **Ask the user** using AskUserQuestion:
   ```
   Question: "Migration failed at [step]. How would you like to proceed?"
   Header: "Error"
   Options:
   - "Rollback changes (git stash)" - Stash all changes and restore original state
   - "Continue debugging" - Keep changes and investigate the issue
   - "Abort" - Stop without rollback (manual cleanup needed)
   ```

2. **If rollback selected**:
   ```bash
   cd "<skill-path>"
   git stash push -m "Failed migration rollback"
   git checkout .
   ```

3. **Report** the specific error and step where it occurred.

|----------|----------|

| File | Action |
|------|--------|
| Rule files | Move (content preserved) |
| SKILL.md | Update links |
| AGENTS.md | Regenerate |
| metadata.json | No change |
| README.md | No change |
