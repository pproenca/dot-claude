# Foundations Checklist

> Base path: `references/foundations/` (relative to the `apple-hig` skill root)

Apply these 7 design foundations after screen layout is defined. Work through them in order — each builds on the previous.

## 1. Typography
**Reference:** `design-typography.md`
**Decide:** Type scale, text styles for headings/body/captions, dynamic type support
**Key question:** What hierarchy of information exists on each screen?

## 2. Writing
**Reference:** `design-writing.md`
**Decide:** Voice and tone, label style (concise vs descriptive), button language, error message style
**Key question:** How does the app speak to users?

## 3. Branding
**Reference:** `design-branding.md`
**Decide:** How brand identity appears without overwhelming the UI — app icon, accent color, custom fonts if any
**Key question:** Where does brand identity enhance vs. distract?

## 4. Color
**Reference:** `design-color.md` + `design-dark-mode.md`
**Decide:** Accent/tint color, semantic colors for status, dark mode approach, contrast requirements
**Key question:** What colors carry meaning vs. decoration?

## 5. Materials
**Reference:** `design-materials.md`
**Decide:** Where to use translucency, vibrancy, background materials — typically tab bars, toolbars, sidebars
**Key question:** Where should the UI feel layered vs. solid?

## 6. Layout
**Reference:** `design-layout.md`
**Decide:** Spacing system, margins, safe areas, how screens adapt across iPhone/iPad/Mac sizes
**Key question:** How does each screen adapt to different screen sizes?

## 7. Motion
**Reference:** `design-motion.md`
**Decide:** Transition style between screens, feedback animations, loading animations
**Key question:** Where does motion add meaning vs. delay?
